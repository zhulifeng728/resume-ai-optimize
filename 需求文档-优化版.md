# ResumeAI — 智能简历反向优化平台

## 产品需求文档（PRD）v1.0

---

## 一、产品概述

### 1.1 产品名称
**ResumeAI**（智简）

### 1.2 产品定位
一个以**真实招聘信息驱动、AI大模型赋能**的一站式求职优化平台。通过聚合主流招聘平台的岗位数据，反向指导用户简历优化，并基于最终简历自动生成技术复习文档，打通**"岗位分析 → 简历优化 → 面试准备"**完整求职闭环。

### 1.3 目标用户
| 用户类型 | 描述 |
|---------|------|
| 应届毕业生 | 缺乏简历撰写经验，不知道如何匹配岗位需求 |
| 跳槽求职者 | 有工作经验，需要针对目标岗位定向优化简历 |
| 转行求职者 | 需要将已有经历重新包装，匹配新行业/岗位要求 |
| 自由职业者/兼职者 | 需要快速生成多版本简历投递不同岗位 |

### 1.4 核心差异化

| 对比维度 | 传统简历工具（超级简历/智联等） | 通用AI工具（ChatGPT/Kimi等） | **ResumeAI（本产品）** |
|---------|---|---|---|
| 优化依据 | 通用模板 + 规则建议 | 用户手动粘贴JD提问 | **系统聚合真实招聘数据，自动关联优化** |
| JD匹配 | 无或需手动对比 | 用户自行操作 | **一键选择目标岗位，AI自动差异分析并改写** |
| ATS适配 | 部分支持 | 无 | **内置ATS关键词评分与优化建议** |
| 面试准备 | 无 | 用户自行提问 | **一键生成结构化技术复习文档** |
| AI模型自由度 | 平台内置，不可选择 | 取决于你用的工具 | **支持BYOK，用户自选模型，不满意随时换** |
| 数据时效性 | 静态模板 | 取决于用户输入 | **实时爬取招聘数据，紧跟市场变化** |

#### 1.2 岗位检索与筛选

| 功能点 | 详细说明 | 优先级 |
|--------|---------|--------|
| 多维筛选 | 按城市、行业、岗位类型、薪资范围、学历要求、经验要求筛选 | P0 |

### 模块4：AI模型与API Key管理（BYOK - Bring Your Own Key）

> **设计理念**：平台提供基础AI能力（使用平台API Key），但由于AI接口调用成本高、免费额度有限，鼓励用户自带API Key（BYOK模式）。这样用户既能享受更充足的调用次数，又能自由选择自己偏好的AI模型，获得更智能的优化体验。

#### 4.1 AI调用层级

| 层级 | 说明 | 适用场景 |
|------|------|---------|
| **平台默认** | 使用平台自有的 NVIDIA API Key（免费额度），为用户提供开箱即用的基础体验 | 新用户试用、轻度使用 |
| **用户自带Key** | 用户配置自己的 API Key，调用次数和模型选择完全由用户自主控制 | 重度使用、追求更优AI质量 |

#### 4.2 支持的模型供应商

| 供应商 | 模型示例 | 特点 | 推荐指数 |
|--------|---------|------|---------|
| **NVIDIA NIM** | Llama 3.1, Mistral, DeepSeek 等 | 注册即送免费额度，成本极低，**平台默认推荐** | ★★★★★ |
| OpenAI | GPT-4o, GPT-4o-mini | 综合能力强，英文优化效果好 | ★★★★ |
| Anthropic | Claude 3.5/4 | 长文本理解强，简历分析细致 | ★★★★ |
| DeepSeek | DeepSeek-V3, DeepSeek-R1 | 中文能力优秀，性价比高 | ★★★★ |
| Google | Gemini 2.0 | 多模态能力，可识别图片简历 | ★★★ |
| 智谱AI | GLM-4 | 国产模型，中文场景友好 | ★★★ |
| 硅基流动/零一万物等 | 各类开源模型 | 价格低廉，可选模型多 | ★★★ |

> **扩展性**：系统采用统一的 LLM 适配层，新模型供应商只需实现标准接口即可接入，用户也可填入任何兼容 OpenAI API 格式的自定义 Endpoint。

#### 4.3 API Key管理功能

| 功能点 | 详细说明 | 优先级 |
|--------|---------|--------|
| Key录入 | 用户在"设置"页面输入自己的API Key，选择对应的供应商和模型 | P0 |
| Key验证 | 录入后系统立即发送测试请求，验证Key是否有效 | P0 |
| Key加密存储 | API Key使用 AES-256 加密后存储，数据库中不保存明文 | P0 |
| 一键删除 | 用户可随时一键删除已存储的Key，删除后系统立即清除所有相关数据，不可恢复 | P0 |
| 模型切换 | 用户可在优化过程中自由切换不同模型，对比不同模型的优化效果 | P1 |
| 用量提示 | 展示当前Key的大致调用消耗（token数），帮助用户了解用量 | P1 |
| 多Key管理 | 支持同时配置多个供应商的Key，按需切换 | P2 |

#### 4.4 Key安全与隐私保障

```
┌─────────────────────────────────────────────────────────────┐
│                    API Key 安全承诺                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 加密存储：所有Key使用AES-256加密，数据库无明文           │
│  2. 最小权限：Key仅用于调用AI接口，不用于任何其他用途        │
│  3. 不留存：用户删除Key后，系统立即彻底清除，无备份          │
│  4. 不共享：用户的Key绝不会被其他用户或平台自身使用          │
│  5. 传输安全：全程HTTPS加密传输，前端不缓存Key明文           │
│  6. 用户可控：用户也可随时在模型供应商官网删除/重置Key       │
│                                                             │
│  💡 建议：用完即删，或在供应商官网设置Key的调用额度上限      │
└─────────────────────────────────────────────────────────────┘
```

## 六、商业模式

| 版本 | 价格 | 权益 |
|------|------|------|
| 免费版 | ¥0 | 平台赠送少量AI调用额度，基础模板，基础复习文档 |
| BYOK模式 | ¥0 | 用户自带API Key，**不限次数AI优化**，全部模板，完整复习文档，ATS评分 |
| 专业版 | ¥29/月 或 ¥199/年 | 使用平台AI额度不限次优化，全部模板，完整复习文档，ATS评分 |
| 终身版 | ¥399 一次性 | 专业版全部权益，终身使用 |

> **核心策略**：BYOK模式让用户零成本获得完整功能体验，降低付费门槛。用户自带Key的调用成本由用户自行承担（NVIDIA免费额度足够个人使用），平台几乎零AI成本。专业版面向不愿自行管理Key的用户，提供"省心"付费方案。

**其他收入**：
- 企业版/API接口（HR批量优化候选人简历）
- 简历模板市场（设计师上传付费模板）

## 九、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 爬虫被封/法律风险 | 数据源断裂 | 优先用户自录入JD作为核心方案，爬虫作为增值补充；研究平台开放API |
| AI生成质量不稳定 | 用户体验差 | 建立Prompt模板库 + 人工审核机制 + 用户反馈闭环优化 |
| 竞品壁垒 | 用户选择已有工具 | 强化"反向优化+复习文档"差异化，快速迭代 |
| AI接口成本高 | 商业模式不可持续 | **BYOK模式为核心策略**：引导用户自带Key（推荐NVIDIA免费额度），平台AI成本趋近于零；付费版面向不愿管理Key的用户 |
| API Key泄露风险 | 用户信任危机 | AES-256加密存储 + HTTPS传输 + 不留存策略 + 引导用户设置额度上限和用后删除 |
